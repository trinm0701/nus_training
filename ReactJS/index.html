useState Hook

State là bộ nhớ riêng của component qua mỗi lần render

useState(initialState) trả về cặp [state, setState]. Khi gọi setState, React lên lịch render lại component với giá trị mới. Giá trị state trong đoạn code đang chạy hiện tại không đổi ngay lập tức.

Công thức để đọc useState

import useState from 'react' const [count, setCount] = useState(0)
  • count: giá trị state trong lần render hiện tại.
  • setCount: function yêu cầu React render lại với giá trị mới.
  • initialState: chỉ dùng ở lần render đầu, có thể là giá trị hoặc initializer function.
  • Updater: dùng setCount(c => c + 1) khi tính từ state trước đó.
import { useState } from 'react';

export default function Counter() {
  const [count, setCount] = useState(0);

  function handleAdd() {
    setCount(count + 1);
  }

  return <button onClick={handleAdd}>{count}</button>;
}

Core idea

React có thể “nhớ” state giữa các lần gọi component như thế nào?

Đây là bản mô phỏng đơn giản để hiểu tư duy của Hook, không phải source code thật của React. Ý chính: React giữ một danh sách state theo thứ tự gọi Hook; mỗi lần render lại, chỉ số Hook quay về 0. 

// Mô phỏng cực giản ý tưởng phía sau useState
let hookStates = [];
let hookIndex = 0;

function useState(initialState) {
  const currentIndex = hookIndex;

  if (hookStates[currentIndex] === undefined) {
    hookStates[currentIndex] =
      typeof initialState === 'function'
        ? initialState()
        : initialState;
  }

  function setState(nextState) {
    const currentState = hookStates[currentIndex];

    hookStates[currentIndex] =
      typeof nextState === 'function'
        ? nextState(currentState)
        : nextState;

    render();
  }

  hookIndex += 1;
  return [hookStates[currentIndex], setState];
}

function render() {
  hookIndex = 0;
  App();
}

function App() {
  const [count, setCount] = useState(0);
  const [name, setName] = useState('React');

  console.log({ count, name });
}

5 lưu ý quan trọng

Những lỗi useState hay gặp trong dự án thật

Khi code React bắt đầu có form, danh sách, filter hoặc checkout, phần lớn bug state đến từ 5 nhóm dưới đây.

1

Chỉ gọi useState ở top level

Gọi Hook ở cấp cao nhất của component hoặc trong Custom Hook. Không đặt useState trong if, vòng lặp hoặc hàm lồng nhau. 

// Sai
if (isAdmin) {
  const [role, setRole] = useState('admin');
}

// Đúng
const [role, setRole] = useState('guest');
const canEdit = role === 'admin';
2

Không thay đổi state trực tiếp

Gán lại biến state như count = count + 1 không báo cho React render lại. Luôn dùng setter mà useState trả về. 

// Sai
count = count + 1;

// Đúng
setCount(count + 1);
3

Dùng updater khi dựa trên state cũ

Nếu next state phụ thuộc previous state, dùng callback để tránh lỗi khi React gộp nhiều update trong cùng một event. 

// Dễ sai khi gọi nhiều lần
setCount(count + 1);

// An toàn hơn
setCount(prevCount => prevCount + 1);
4

State không đổi ngay trong hàm hiện tại

Sau khi gọi setter, biến state hiện tại vẫn giữ giá trị của render đang chạy. Giá trị mới chỉ xuất hiện ở lần render tiếp theo. 


const [count, setCount] = useState(0);

function handleClick() {
  const nextCount = count + 1;
  setCount(nextCount);
  console.log(count);     // ??
  console.log(nextCount); // ??
}
5

Object và array cần reference mới

Không sửa thuộc tính bên trong object/array rồi truyền lại chính object cũ. Hãy tạo bản sao mới bằng spread, map hoặc filter. 

// Sai
user.name = 'John';
setUser(user);

// Đúng
setUser({
  ...user,
  name: 'John'
});

Event handling

Event là cầu nối giữa người dùng và state

Trong JSX, event prop viết dạng camelCase như onClick, onChange. Giá trị của prop phải là function để React gọi sau, không phải kết quả của function.

Truyền function: đúng

function handleClick() {
  alert('Clicked');
}

<button onClick={handleClick}>
  Save
</button>

React ghi nhớ handler và chỉ gọi khi user click.

Gọi function khi render: sai ý

<button onClick={handleClick()}>
  Save
</button>

<button onClick={alert('Clicked')}>
  Save
</button>

Dấu () làm code chạy ngay trong lúc render, trước khi user click.

  const [selectedOrder, setSelectedOrder] = useState(null);

  async function handleConfirm(orderId) {
    await confirmOrder(orderId);

function OrderCard({ order, onViewDetail, onConfirm, onCancel }) {
  return (
    <article>
      <h3>Đơn #{order.code}</h3>
      <button onClick={() => onViewDetail(order)}>Chi tiết</button>
      <button onClick={() => onConfirm(order.id)}>Xác nhận</button>
      <button onClick={() => onCancel(order.id)}>Hủy</button>
    </article>
  );
}

    Sharing State Between Components

    Khi nhiều component cùng phụ thuộc một lựa chọn, đưa state lên cha gần nhất

    Ví dụ thực tế: trong trang checkout, phần chọn phương thức giao hàng, phần dự kiến nhận hàng và phần tổng tiền đều cần biết cùng một lựa chọn. Nếu mỗi component tự giữ state riêng, UI rất dễ lệch nhau. Cách React khuyên dùng là để component cha gần nhất giữ source of truth.

    3 bước lifting state up trong checkout

    1. Tìm state đang bị lặp: shippingId xuất hiện ở nhiều component.
    2. Đưa state lên CheckoutPage, cha chung của các phần đó.
    3. Truyền selectedShipping xuống để hiển thị và onSelectShipping xuống để child báo thay đổi.
    function CheckoutPage() {
  const [shippingId, setShippingId] = useState('standard');
  const shipping = shippingMethods.find(m => m.id === shippingId);

  return (
    <>

        onSelect={setShippingId}
      />
      <DeliveryPreview shipping={shipping} />
      <OrderSummary shippingFee={shipping.fee} />
    </>
  );
}

    Controlled components

    Input controlled là input bị state điều khiển

    Forms

    Form gồm nhập liệu, validate và submit

    Với React hiện đại, bạn có hai cách hay gặp: xử lý onSubmit và đọc new FormData(event.currentTarget), hoặc dùng React DOM <form action={fn}> để function nhận trực tiếp formData.

    Uncontrolled + FormData

    DOM giữ giá trị input. Khi submit, event.currentTarget chính là thẻ <form> đang được submit; FormData sẽ đọc các field có name bên trong form đó. 

    function handleSubmit(event) {
  event.preventDefault();

  const formData = new FormData(event.currentTarget);
  const data = {
    student: formData.get('student'),
    email: formData.get('email'),
    topic: formData.get('topic')
  };

  registerStudent(data);
}

<form onSubmit={handleSubmit}>
  <input name="student" />
  <input name="email" type="email" />
  <select name="topic">...</select>
  <button type="submit">Gửi</button>
</form>

    React DOM action function

    Với <form action={fn}>, React gọi function đó khi submit và truyền trực tiếp formData. Trường hợp này không cần tự tạo new FormData(event.currentTarget) trong handler. 

    function signup(formData) {
  const data = {
    student: formData.get('student'),
    email: formData.get('email'),
    topic: formData.get('topic')
  };

  registerStudent(data);
}

<form action={signup}>
  <input name="student" />
  <input name="email" type="email" />
  <select name="topic">...</select>
  <button type="submit">Gửi</button>
</form>

    Form lab

    Submit form và đọc FormData

    Khi dùng onSubmit, gọi event.preventDefault() để chặn trình duyệt submit theo cách mặc định. Nếu không chặn, browser sẽ reload hoặc chuyển trang, làm mất state hiện tại và React không kịp xử lý dữ liệu trong hàm submit.

    Tips

    Checklist khi viết state, events và forms

    State

    Dùng updater khi next state phụ thuộc previous state, đặc biệt nếu gọi set nhiều lần trong một event.

    Object/array

    Không sửa trực tiếp object/array trong state. Tạo bản mới bằng spread, map, filter hoặc reducer.

    Events

    Đặt handler là function: onClick={handleClick} hoặc onClick={() => doThing(id)}.

    Props handler

    Với component tự tạo, đặt prop theo ý nghĩa: onSubmitOrder, onSelectUser.

    Sharing

    Khi hai child cần đồng bộ, tìm cha chung gần nhất và để cha giữ source of truth.

    Controlled input

    Text input cần value + onChange; checkbox cần checked + onChange.

    Submit

    Nếu dùng onSubmit, gọi event.preventDefault() để tránh browser refresh trang.

    Accessibility

    Mỗi input nên có label rõ ràng. Button submit nên có nội dung cụ thể hơn "OK".
